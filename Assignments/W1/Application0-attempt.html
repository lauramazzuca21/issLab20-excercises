Requirements

Design and build a software system to control a DDR (differential drive robot) so that:
• The robot is able to explore in a sistematic and autonomus way a room
• By showing (in real time) a picture of the explored space

Requirement analysis

DDR: Differential Drive Robot; two-wheeled drive system with independent actuators for each wheel.
room: a defined and confined space in which the DDR can move.
sistematic: the DDR will move in the room following a specific pattern of exploration to cover it all...
autonomus: ... and it will do so without requiring input from a user.
showing real time a picture: the DDR will need to send on a given device (computer, smartphone) images of its exploration.
Functional TestPlan
• the Software System will need to ensure the full coverage of the room and avoidance of obstacles.
• the Software System will need to ensure the communication between the DDR and the device that will need to receive the .

Problem analysis

Logical Architecture: To satisfy the requirement of exploration the Software System will need a module to drive the DDR, which means making decisions on the direction where to go depending on the detections of the sensors. Then, there will also be a need for a module to manage the connection with the device that will receinve the pictures, which will need to establish the connection and send the pictures taken by the camera.
Abstraction Gap: the gap will have to be consistent so that it's easier to build a system easily extensible and maintenable.
